# Contributing

## Branching

- Create a feature branch from `main`.
- Use clear branch names such as `feature/workflow-engine` or `fix/auth-session`.

## Commits

- Keep commits focused.
- Write descriptive commit messages.

## Pull Requests

- Explain the problem being solved.
- Summarize the technical approach.
- Include screenshots for UI changes.
- Mention any architectural impact.

## Code Expectations

- Use TypeScript where possible.
- Prefer shared packages for reusable types and config.
- Keep modules loosely coupled.
- Add tests for workflow logic and critical services.
